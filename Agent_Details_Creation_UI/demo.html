How to Use AgentPortal

Complete step-by-step guide to filling out your company profile and importing your product data efficiently

Step 1: Company Information

1Basic Details

Start by filling out your company's basic information:

  • Company Name: Your official business name
  • Establishment Date: When your company was founded
  • Owner Name: Full name of the business owner
  • Customer Service: Your support phone number
Example:
Company: "TechSolutions Inc."
Established: "2020-01-15"
Owner: "John Smith"
Phone: "+1 (555) 123-4567"

2Company Description

Write a clear description of what your company does. This helps us understand your business better.

Good Example:
"We provide cloud-based software solutions for small and medium businesses, specializing in inventory management and customer relationship tools that help streamline operations and increase productivity."

Pro Tips:

  • Keep it concise but descriptive
  • Focus on your main services/products
  • Mention your target audience
  • Highlight your unique value proposition

Step 3: File Format Examples

TXTText File Format

For TXT files, list each product/service on a separate line. You can include basic details separated by commas or pipe symbols.

Item Name: Customer Data Processing Suite
Item Code: CDPS-2025

Initial Working Explanation: This solution automates the process of uploading, validating, cleaning, and analyzing customer sales data. It generates detailed reports, detects anomalies, and exports results in multiple formats.

Implementation Steps:
step1: Upload the raw sales data file
step2: Verify all mandatory columns (Date, Customer, Product, Amount)
step3: Clean invalid entries and handle missing values
step4: Process data to generate monthly summaries
step5: Apply business rules to detect anomalies
step6: Create visual reports (charts & dashboards)
step7: Export the final report to PDF/Excel
step8: Share the results with the client

Common Issues & Solutions:
issue1: Data upload fails due to wrong format
solution1: Ensure file is in CSV/Excel and headers match template

issue2: Report generation takes too long
solution2: Optimize dataset by filtering unnecessary columns

issue3: Exported report missing charts
solution3: Reinstall chart rendering library and retry export

issue4: Duplicate entries in final output
solution4: Apply deduplication rules before processing

TXT File Tips:

  • Use "Item Name:" and "Item Code:" headers
  • Include "Initial Working Explanation:" section
  • List "Implementation Steps:" with step1, step2, etc.
  • Add "Common Issues & Solutions:" with issue/solution pairs
  • Use clear formatting with proper line breaks

JSONJSON File Format

JSON format allows for more structured data with detailed information about each product or service.

{
  "Item": {
    "Name": "Customer Data Processing Suite",
    "Code": "CDPS-2025",
    "Description": "This solution automates the process of uploading, validating, cleaning, and analyzing customer sales data. It generates detailed reports, detects anomalies, and exports results in multiple formats."
  },

    {"step": 4, "description": "Process data to generate monthly summaries"},
    {"step": 5, "description": "Apply business rules to detect anomalies"},
    {"step": 6, "description": "Create visual reports (charts & dashboards)"},
    {"step": 7, "description": "Export the final report to PDF/Excel"},
    {"step": 8, "description": "Share the results with the client"}
  ],
  "CommonIssuesAndSolutions": [
    {
      "issue": "Data upload fails due to wrong format",
      "solution": "Ensure file is in CSV/Excel and headers match template"
    },
    {
      "issue": "Report generation takes too long",
      "solution": "Optimize dataset by filtering unnecessary columns"
    },
    {
      "issue": "Exported report missing charts",
      "solution": "Reinstall chart rendering library and retry export"
    },
    {
      "issue": "Duplicate entries in final output",
      "solution": "Apply deduplication rules before processing"
    }
  ]
}

JSON Structure Tips:

  • Use "Item" object with "Name", "Code", and "Description"
  • Include "ImplementationSteps" array with step numbers
  • Add "CommonIssuesAndSolutions" with issue/solution pairs
  • Ensure proper JSON formatting and validation
  • Each step should have "step" number and "description"
